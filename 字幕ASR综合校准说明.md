# 字幕 ASR 综合校准说明

## 为什么需要校准

市面上大多数视频翻译产品只依赖单一文本来源——要么 ASR（语音识别），要么 SRT（字幕文件），要么 OCR（画面文字识别）。单一来源各有缺陷：

- **ASR**：方言、口音、背景音乐干扰，识别结果常有错误
- **SRT**：字幕本身也可能有错别字、遗漏、时间轴偏移
- **OCR**：分辨率低、字幕特效干扰，识别率不稳定

VideoTrans 采用 **ASR + SRT 综合校准**策略——同时参考两种来源，交叉验证，取长补短，生成比任一单一来源都更准确的文本。

![综合ASR校准SRT错误例子](resourses/综合ASR校准SRT错误例子.jpg)

上图中，ASR 将"女娃子"误识别为"女孩子"，而 SRT 正确；同时 SRT 将"排着队想嫁给你"拆成了两段且有遗漏，ASR 却完整识别。综合校准能从两个来源中各取正确部分，产出最优结果。

---

## 校准的核心难点：多对多对齐

ASR 和 SRT 的句子并非一一对应，而是**多对多**的关系：

- 一句 SRT 可能对应多句 ASR（SRT 是长句，ASR 切分更细）
- 一句 ASR 可能对应多句 SRT（ASR 合并了短句）
- 某些句子只有 ASR 有、SRT 没有，或反之

更复杂的是，当视频中存在**背景旁白**（如片头曲、片尾曲、画外音）时，字幕往往无法与 ASR 对应。以下案例来自一部真实短剧的校准日志（[完整日志](resourses/sentence-reconcile-log.md)）。

### 案例 1：方言纠错——ASR 和 SRT 互为校正

| | 时间范围 | 文本 |
|--|----------|------|
| ASR | 00:16.473 ~ 00:17.762 | 莫浪费时间，**就咱有钱**。 |
| SRT | 00:16.473 ~ 00:17.762 | 莫浪费时间，**救灾要紧** ⇥ 救灾要紧 |
| **校准后** | | 莫浪费时间，**救灾要紧**。 |

ASR 将方言"救灾要紧"误听成"就咱有钱"，SRT 给出了正确文本。

| | 时间范围 | 文本 |
|--|----------|------|
| ASR | 02:35.923 ~ 02:38.651 | 好多的**女孩子**排着队想嫁给你。 |
| SRT | 02:35.923 ~ 02:38.651 | 你晓不晓得 ⇥ 好多的**女娃子**排着队 ⇥ 想嫁给你 |
| **校准后** | | 好多的**女娃子**排着队想嫁给你。 |

ASR 将四川方言"女娃子"误听成"女孩子"，SRT 正确。

### 案例 2：背景歌曲误识别

这是最复杂的场景——对白与背景歌曲交织，ASR 和 SRT 在歌曲部分完全对不上：

**ASR 识别（5 句）：**

| 时间范围 | ASR 文本 |
|----------|----------|
| 02:40.953 ~ 02:47.489 | 点燃一盏灯的暖黄，照亮夕阳穿过我四方。 |
| 02:47.493 ~ 02:48.387 | 哥哥。 |
| 02:48.386 ~ 02:50.174 | 木啊木啊。 |
| 02:50.173 ~ 02:52.483 | 我为你哭。 |
| 02:52.480 ~ 02:58.250 | 哎呀，我跟你们说嘛，今天接生婆已经来摸过了，说的是个男娃儿。 |

**对应 SRT（5 条）：**

| 时间范围 | SRT 文本 |
|----------|----------|
| 163.842 ~ 168.758 | 小城故事多 |
| 168.875 ~ 172.842 | 充满喜和乐 |
| 172.958 ~ 174.158 | 哎呀我跟你们说嘛 |
| 174.275 ~ 176.442 | 今天接生婆已经来摸过了 |
| 176.558 ~ 177.875 | 说这是个男娃儿 |

**分析：**

1. **对白部分**（后 3 条 SRT）：ASR 的最后一句涵盖了 SRT 的后 3 条，但 ASR 把"说这是个男娃儿"误识别为"说的是个男娃儿"——差异很小，校准后采纳 SRT 的"说这是个男娃儿"
2. **背景歌曲**（前 2 条 SRT）："小城故事多 / 充满喜和乐"是背景歌曲，SRT 正确记录了歌词，但 ASR 完全误听成了"点燃一盏灯的暖黄，照亮夕阳穿过我四方"——典型的 ASR 对音乐歌词识别不准
3. **ASR 幻觉**：ASR 在歌曲期间还产生了"哥哥""木啊木啊""我为你哭"等幻觉内容，与实际歌词完全无关

**校准结果：**

| 时间范围 | 校准前（ASR） | 校准后 |
|----------|---------------|--------|
| 02:40.953 ~ 02:47.489 | 点燃一盏灯的暖黄，照亮夕阳穿过我四方。 | 小城故事多， |
| 02:47.493 ~ 02:48.387 | 哥哥。 | 充满喜和乐。 |
| 02:48.386 ~ 02:50.174 | 木啊木啊。 | 哎呀我跟你们说嘛， |
| 02:50.173 ~ 02:52.483 | 我为你哭。 | 今天接生婆已经来摸过了， |
| 02:52.480 ~ 02:58.250 | 哎呀，我跟你们说嘛，今天接生婆已经来摸过了，说的是个男娃儿。 | 说这是个男娃儿。 |

背景歌曲部分完全依赖 SRT 纠正，对白部分则综合两者的最佳结果。

> 💡 目前人声分离模型尚未针对歌声做专项剔除，背景歌曲会混入人声轨道干扰 ASR。后续计划在人声分离后增加歌唱声分离，进一步降低此类干扰。

### 案例 3：SRT 孤岛——ASR 完全漏听的句子

| SRT 索引 | 时间范围 | SRT 文本 |
|----------|----------|----------|
| 26 | 00:48.042 ~ 00:49.358 | 轻点轻点 |

这句 SRT 没有任何 ASR 句子覆盖（"孤岛"），可能是音量过小或语速过快导致 ASR 完全漏听。校准系统会将这类孤岛 SRT 直接保留，避免内容丢失。

## 校准日志

每次翻译的校准过程都会生成详细日志，保存在 `segments/ASR/sentence-reconcile-log.md` 中，包含：

- **总览表**：每组的 ASR/SRT 对齐关系、冲突类型
- **校准前后对比总表**：逐句对比，标注修改状态和来源（LLM 校准 / 预匹配 / SRT 直通）
- **孤岛 SRT 句子**：ASR 未覆盖的 SRT 内容
