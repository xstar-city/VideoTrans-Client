# 批量视频翻译 · 断点续跑说明

## 背景

一部短剧通常包含 60+ 个视频文件，一次批量翻译任务可能运行数小时。运行过程中可能遇到：

- 需要中途取消（按 `Ctrl+C`）
- 网络中断导致客户端脚本意外退出
- 电脑关机/休眠

**好消息：这些情况都不需要从头重跑。** 服务端支持断点续跑——已完成的步骤（ASR、翻译、TTS 等）会自动跳过，只执行之前未完成的部分。

---

## 两种翻译方式

### 1. 单视频 / 少量视频翻译

使用 `video_translate.py`，直接指定视频文件路径：

```bash
# 翻译单个视频
python video_translate.py "E:\短剧\《逐玉》\1\1.mp4" -t en --server http://<ServerIP>:8000

# 翻译多个视频（需在同一次命令中指定）
python video_translate.py "E:\短剧\《逐玉》\1\1.mp4" "E:\短剧\《逐玉》\2\2.mp4" -t en --server http://<ServerIP>:8000
```

### 2. 整部短剧批量翻译

使用 `short_drama_translate.py`，指定短剧根目录，自动扫描所有视频：

```bash
python short_drama_translate.py "E:\短剧\《逐玉》" -t en --server http://<ServerIP>:8000
```

脚本会自动：
- 整理视频到标准目录结构
- 转换非 mp4 格式
- 检测哪些视频已有翻译结果（只翻译缺失的）
- 按目标语言分组批量处理

---

## 如何续跑

非常简单：**重新运行和之前完全一样的命令即可。**

```bash
# 第一次运行（中途退出了）
python short_drama_translate.py "E:\短剧\《逐玉》" -t en --server http://<ServerIP>:8000

# 续跑：执行同样的命令
python short_drama_translate.py "E:\短剧\《逐玉》" -t en --server http://<ServerIP>:8000
```

```bash
# video_translate 同理
python video_translate.py "E:\短剧\《逐玉》\1\1.mp4" -t en --server http://<ServerIP>:8000
# 续跑：同样的命令再跑一次
python video_translate.py "E:\短剧\《逐玉》\1\1.mp4" -t en --server http://<ServerIP>:8000
```

脚本会自动检测到之前的任务，复用服务端的工作目录，跳过已完成的步骤，从断点继续。

---

## 目录要求

**每个视频必须放在独立的文件夹中。** 这是项目的基本约定，确保翻译过程中生成的中间文件（ASR 结果、翻译文本、TTS 音频等）不会互相覆盖。

```
正确 ✅
E:\短剧\
├── 《逐玉》\
│   ├── 1\
│   │   └── 1.mp4
│   ├── 2\
│   │   └── 2.mp4
│   └── 3\
│       └── 3.mp4
└── 《花间》\
    ├── 1\
    │   └── 1.mp4
    └── 2\
        └── 2.mp4

错误 ❌
E:\短剧\
├── 逐玉_1.mp4
├── 逐玉_2.mp4
├── 花间_1.mp4      ← 不同剧的视频混在同一目录，文件会互相覆盖
└── 花间_2.mp4
```

> 💡 使用 `short_drama_translate.py` 时，脚本会自动整理视频到标准目录结构，无需手动操作。

---

## `.vt_task_id` 文件的作用

运行翻译任务后，每个视频所在的文件夹下会自动生成一个 `.vt_task_id` 隐藏文件，记录本次任务的编号。

这个文件是断点续跑的关键：

- **续跑时**：脚本读取该文件，找到服务端上对应的工作目录，继续未完成的步骤
- **任务完成后**：该文件会自动删除

> 💡 如果在资源管理器中看不到 `.vt_task_id` 文件，需要开启"显示隐藏文件"选项。

---

## 强制重新开始

如果不想续跑，想完全从零开始，有两种方式：

1. **删除 `.vt_task_id` 文件**：手动删除视频目录下的 `.vt_task_id` 文件，再重新运行命令
2. **使用 `--new-task` 参数**：运行时加 `--new-task`，会忽略旧任务，创建全新任务

```bash
python video_translate.py "E:\短剧\《逐玉》\1\1.mp4" -t en --server http://<ServerIP>:8000 --new-task
```

> ⚠️ 强制重新开始会创建新的服务端任务，之前已完成的步骤不会自动复用（但服务端工作目录中的缓存文件仍可被流水线识别跳过）。

---

## 多视频任务的注意事项

批量翻译多个视频时，所有视频必须在**同一次任务**中一起运行。脚本会校验：

- 所有视频目录下必须**都没有** `.vt_task_id`（全新任务）
- 或者所有视频目录下的 `.vt_task_id` **完全一致**（续跑同一任务）

如果出现不一致（例如某些视频曾单独跑过，某些没有），脚本会报错提示，避免文件混乱。此时请删除相关目录下的 `.vt_task_id` 文件后重新运行。

## 常见问题

**Q: 续跑时可以修改目标语言吗？**  
A: 不建议。续跑复用的是同一个服务端工作目录，修改参数可能导致结果不一致。如需修改，请用 `--new-task` 创建新任务。

**Q: 客户端脚本退出后，服务端还在跑吗？**  
A: 取决于退出方式。按 `Ctrl+C` 会通知服务端取消任务；如果是网络断开，服务端任务仍会继续执行到当前步骤完成。两种情况都可以续跑。

**Q: 续跑时上传会重复吗？**  
A: 不会。脚本会自动检查服务端已有文件，大小一致则跳过上传。

**Q: short_drama_translate.py 中途退出后再跑，会重新整理视频吗？**  
A: 会，但整理操作是幂等的——已整理好的视频不会被重复处理。已翻译完成的视频也会自动跳过。
