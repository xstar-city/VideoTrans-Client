# 客户端可见文件结构说明

运行音频翻译后，任务工作目录中**客户端可下载**的文件结构如下。
不可见的内部文件（`.json`、`.wav`、`.csv`、调试日志等）不在此列。

```text
<任务工作目录>/
├── input.mp3                          ← 输入音频（原始文件，不会被修改）
├── input_vocals_denoised.mp3          ← 分离出的人声（经降噪，仅启用人声分离时生成）
├── input_others.mp3                   ← 分离出的背景音（始终未降噪，保留背景细节）
├── input.srt                          ← 原始字幕（如有）
├── non_speech_vocal_events/           ← 检测到的非语言人声事件（笑声、咳嗽、唱歌等）
│   ├── 0001_laughter_00000000_00005000.mp3   事件切片音频（可试听下载）
│   ├── 0002_singing_00008000_00015000.mp3   文件名 = 序号_类型_起始毫秒_结束毫秒
│   └── ...
└── segments/                          ← 所有翻译产出
    ├── ASR/                           ← 语音识别结果
    │   ├── full_text.md                  全文识别文本（所有逐段 txt 合并）
    │   ├── 0.000.txt                     逐段识别文本
    │   ├── 1.520.txt                     文件名 = 起始秒数（三位小数）
    │   ├── 3.840.txt                     可用于校对原始识别内容
    │   ├── ...
    │   ├── sentence-reconcile-log.md     句子校准日志（识别→字幕校准过程）
    │   └── secondary-diarization-        二次说话人切分校准日志（含短句合并）
    │       calibration-log.md            （仅 --asr-mode precise）
    │
    ├── 0.000.mp3                      ← 切分的源音频片段（与 ASR txt 一一对应）
    ├── 1.520.mp3
    ├── 3.840.mp3
    ├── ...
    │
    ├── full_text_en.txt               ← 英文翻译全文
    ├── translation_guidelines.txt     ← 翻译指南（本次使用的翻译要求）
    │
    └── English/                       ← 目标语言目录（每个目标语言一个）
        ├── 0.000.txt                     逐段翻译文本（文件名 = 起始秒数，三位小数）
        ├── 1.520.txt                     可直接编辑后重新合成
        ├── ...
        ├── 0.000.md                      翻译过程记录
        ├── 1.520.md                      （包含原文、初译、终译的对比）
        ├── ...
        ├── 0.000.mp3                     逐段合成音频（文件名 = 起始秒数，三位小数）
        ├── 1.520.mp3                     （与翻译文本一一对应）
        ├── ...
        ├── 0.000/                        该片段的 TTS 合格候选音频目录
        │   ├── 1.mp3                        候选 1
        │   ├── 2.mp3                        候选 2
        │   ├── 3.mp3                        候选 3
        │   └── candidates.md                候选说明：文本、音色相似度、时长信息
        ├── combined.mp3                ← 合并后的完整合成人声
        ├── final.mp3                   ← 最终输出音轨（含背景音时为混音）
        ├── full_translation.srt        ← 完整翻译字幕（翻译完成后自动生成，可人工编辑检查）
        └── translation_guidelines.txt  ← 翻译指南（翻译风格/术语要求，可编辑后重跑）
```

## 可见性规则

| 规则 | 说明 |
|------|------|
| 目录通常可见 | 客户端可逐级导航进入子目录（`_tts_refs/` 除外） |
| 语言目录下候选子目录可见 | `segments/{语言}/{秒数}/` 保存该片段的合格候选音频和 `candidates.md`，可下载试听 |
| `.mp3` / `.txt` / `.srt` 始终可见 | 覆盖输入音频、切分片段、字幕、翻译文本、候选音频、非语言人声事件切片等 |
| `segments/ASR/` 下仅允许预定义文档 | `full_text.md`、`sentence-reconcile-log.md`、`secondary-diarization-calibration-log.md` |
| `segments/{语言}/` 下所有文件可见 | TTS 音频、翻译文本、翻译过程记录、候选音频目录等 |
| 隐藏文件不可见 | 以 `.` 开头的文件（如 `.tts_task_info`） |

## 文件说明

### 输入音频与人声分离

| 文件 | 说明 |
|------|------|
| `input.mp3` | 客户端上传的原始音频，不会被修改 |
| `input_vocals_denoised.mp3` | 人声分离后的人声音频（经降噪，仅启用人声分离时生成） |
| `input_others.mp3` | 人声分离后的背景音（**始终未降噪**，降噪模型对音乐/环境声会过度抑制） |
| `input.srt` | 客户端上传的原始字幕文件（如有） |

### 非语言人声事件

| 文件 | 说明 |
|------|------|
| `non_speech_vocal_events/{序号}_{类型}_{起始毫秒}_{结束毫秒}.mp3` | 检测到的非语言人声事件切片（笑声、咳嗽、唱歌、掌声、叹气等），可下载试听 |

> **说明**：系统会自动从人声音频中检测非语言人声事件，包括笑声（Laughter）、咳嗽（Cough）、打喷嚏（Sneeze）、掌声（Applause）、叹气（Sigh）、唱歌（Singing）等。每个事件会裁切出对应的音频片段，转换为 mp3 供客户端下载。这些事件信息用于流水线中自动处理（如唱歌片段会从人声中分离出来叠加到背景音），客户端可试听确认检测准确性。

### 语音识别（ASR）

| 文件 | 说明 |
|------|------|
| `ASR/full_text.md` | 完整的识别文本，按段落排列 |
| `ASR/{秒数}.txt` | 单段识别文本，文件名为该段在原始音频中的起始秒数（三位小数）。第一行为 ASR 识别文本，第二行为对应音频片段的时长（秒，三位小数） |
| `ASR/sentence-reconcile-log.md` | 识别结果与字幕校准的过程记录 |
| `ASR/secondary-diarization-calibration-log.md` | 二次说话人切分校准日志，含短句合并过程（仅 `--asr-mode precise`，详见[说明](二次说话人切分校准日志说明.md)） |
| `{秒数}.mp3` | 对应的源音频片段 |

### 翻译

| 文件 | 说明 |
|------|------|
| `full_text_{语言}.txt` | 该语言的完整翻译文本 |
| `translation_guidelines.txt` | 本次使用的翻译指南 |
| `{语言}/{秒数}.txt` | 单段翻译文本，文件名 = 起始秒数（三位小数），可直接编辑 |
| `{语言}/{秒数}.md` | 翻译过程记录，包含原文与译文的对照 |
| `{语言}/full_translation.srt` | 完整翻译字幕（翻译完成后自动生成，可人工编辑检查） |
| `{语言}/translation_guidelines.txt` | 该语言的翻译指南（翻译风格/术语要求，可编辑后重跑） |

### 语音合成与输出

| 文件 | 说明 |
|------|------|
| `{语言}/{秒数}.mp3` | 单段合成音频，文件名 = 起始秒数（三位小数） |
| `{语言}/{秒数}/` | 该片段的 TTS 合格候选音频目录，包含 `1.mp3`、`2.mp3`、`candidates.md` 等，可下载人工试听 |
| `{语言}/combined.mp3` | 合并后的完整合成人声（不含背景音） |
| `{语言}/final.mp3` | **最终输出**：完整目标语言音轨 |

> **注意**：候选目录与正式合成音频同名（如 `0.000/` 对应 `0.000.mp3`）。`candidates.md` 会记录每条候选对应的翻译文本、音色相似度和时长信息，便于人工调试与替换。

## 多目标语言

指定多个目标语言时，会生成多个语言目录：

```text
segments/
├── English/    ← 英文
│   ├── combined.mp3
│   └── final.mp3
├── Hindi/      ← 印地语
│   ├── combined.mp3
│   └── final.mp3
└── ...
```

语言目录名使用语言显示名（如 `English`、`Hindi`），而非短代码（如 `en`、`hi`）。
