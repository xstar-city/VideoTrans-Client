# 错误代码说明

当翻译流水线执行失败时，客户端会显示一个错误代码和简要说明。本文档列出所有可能的错误代码及其含义，帮助您快速定位问题。

## 错误代码列表

| 错误代码 | 说明 | 可能原因 | 建议操作 |
|---------|------|---------|---------|
| `E_INPUT` | 输入文件无法读取或格式不支持 | 文件损坏、格式不支持、文件不存在 | 检查输入文件是否完整，确认格式受支持（mp4/mkv/mov/mp3/wav 等） |
| `E_SEPARATION` | 人声分离失败，无法从音频中分离出人声 | 分离服务未启动、显存不足、音频文件异常 | 检查服务端分离服务是否正常；尝试加 `--no-separate` 跳过分离 |
| `E_ASR` | 语音识别失败，无法将语音转换为文字 | 识别服务不可用、音频质量过差、说话人切分服务异常 | 检查服务端 ASR 服务是否正常；确认音频中包含清晰的人声 |
| `E_TRANSLATION` | 文本翻译失败，翻译模型调用异常导致翻译文本缺失 | 翻译模型服务不可用、API 额度用尽、网络异常 | 检查翻译模型服务是否可用；稍后重试 |
| `E_TTS` | 语音合成失败，多次尝试未找到满足时长和音色要求的候选音频 | 翻译文本过长/过短导致合成音频时长无法匹配原段，或音色相似度不达标 | 检查缺失片段对应的翻译文本是否准确，修正后用 `--edit-rerun` 重跑 |
| `E_COMBINE` | 音频合并失败，无法将分段音频合并为完整音轨 | 音频文件缺失或损坏、磁盘空间不足 | 重试；如持续失败请联系服务端管理员 |
| `E_MIX` | 最终混音失败，无法将人声与背景音混合 | 背景音轨或人声轨缺失、音频格式异常 | 重试；如持续失败请联系服务端管理员 |
| `E_GPU` | GPU 显存不足，输入音视频时长过长导致模型推理时显存溢出 | 输入音视频时长过长，超出单次推理显存容量 | 将长音频/视频拆分为较短的片段分别翻译，或联系服务端管理员升级硬件 |
| `E_MODEL` | 客户端指定了不支持的模型名称 | `--translation-models` 参数中的模型名称拼写错误（如 `doubao-seed-2-0-pro` 误写，正确为 `doubao-seed-2.0-pro`） | 检查模型名称拼写，参考服务端日志中列出的支持的模型列表 |
| `E_CANCEL` | 任务已取消 | 用户主动取消或服务端管理员取消 | 重新运行同样的命令即可续跑 |
| `E_UNKNOWN` | 流水线发生未知错误 | 未预见的异常 | 查看服务端日志获取详细信息，联系服务端管理员 |

## 断点续跑

**所有错误退出后，都可以重新运行同样的命令续跑。** 已完成的步骤会自动跳过，只需重新执行失败的部分。

例如，如果翻译阶段失败（`E_TRANSLATION`），重新运行命令后：
- 人声分离、ASR 等已完成步骤会自动跳过
- 仅重新执行翻译及后续步骤

无需任何额外参数，直接重跑即可。

## 查看详细日志

客户端显示的错误代码为简要说明。如需排查具体原因，可通过以下途径获取详细信息：

| 途径 | 说明 |
|------|------|
| 服务端 Web UI | 浏览器访问 `http://<ServerIP>:8000/`，查看任务历史和状态 |
| 服务端 pipeline.log | 完整的流水线日志，位于任务工作目录下 |
| 服务端 stderr.log | 错误堆栈信息，位于任务工作目录下 |
| 服务端 API 文档 | `http://<ServerIP>:8000/docs`，查看任务状态和输出 |

## 常见问题

**Q: 出现 E_TTS 错误怎么办？**

A: TTS 合成失败通常是因为多次尝试并未找到满足时长和音色要求的候选音频。请尝试：
1. 检查缺失片段对应的翻译文本是否准确（过长或过短都会导致时长无法匹配）
2. 修正翻译文本后，使用 `--edit-rerun` 模式重跑，仅重新合成受影响的片段

**Q: 出现 E_TRANSLATION 错误怎么办？**

A: 翻译失败通常是因为翻译模型服务不可用。请检查：
1. 翻译模型 API 是否可用（额度、网络等）
2. 尝试更换翻译模型（`--translation-models` 参数）
3. 稍后重试，可能是临时性服务波动

**Q: 出现 E_GPU 错误怎么办？**

A: GPU 显存不足通常是因为输入的音视频时长过长，模型推理时显存溢出。由于一个服务端同时只运行一个任务，不会因多任务抢占导致此问题。请尝试：
1. 将长音频/视频拆分为较短的片段（如每段 10-15 分钟），分别翻译
2. 如持续出现，联系服务端管理员确认 GPU 硬件规格是否满足需求

**Q: 出现 E_MODEL 错误怎么办？**

A: 此错误表示 `--translation-models` 参数中包含了不支持的模型名称。常见原因是模型名称拼写错误，例如：
- `doubao-seed-2-0-pro`（错误，用了连字符）-> 正确为 `doubao-seed-2.0-pro`（版本号用点号）
- `deepseek-v4-pro`（正确） vs `deepseek_v4_pro`（错误，用了下划线）

请检查：
1. 服务端日志中会列出当前支持的所有模型名称，以此为准确认拼写
2. 模型名称中版本号部分使用点号（如 `2.0`），不用连字符（如 `2-0`）
3. 修正模型名称后重新运行即可续跑
